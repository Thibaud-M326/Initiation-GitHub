ATELIER M. Cahier des Charges Technique Application de prise de rendez-vous --- Salon d'onglerie Version 1.0 --- Février 2026 Document destiné à Claude Code / Développeur

Vision du Projet 1.1 Description générale Application web de prise de rendez-vous pour un salon d'onglerie haut de gamme. L'application se compose d'une landing page élégante côté client et d'un tableau de bord administrateur pour gérer les rendez-vous, le calendrier et les clients. 1.2 Philosophie Design Ton : Luxe épuré. Classe avant girly. Penser Aesop / boutique parisienne. Palette : Taupe chaud (#8B7355), champagne doré (#C4A87C), crème (#F5F0EB), blanc, noir doux (#2C2C2C) Typographie : Serif élégant pour les titres (ex: Playfair Display, Cormorant Garamond), sans-serif fin pour le corps (ex: Jost, DM Sans) Ambiance : Marbre, mains soignées, lumière douce, beaucoup d'espace blanc Interdit : Rose bonbon, paillettes, emojis ongles, clip-art girly 1.3 Référence visuelle Le mockup de référence (image fournie) montre : Hero plein écran avec photo de mains manucurées sur fond marbre/doux Titre serif centré : « Réservez votre moment » Champ email + bouton CTA doré « Prendre rendez-vous » Widget calendrier intégré (sélection date + créneaux horaires) Section inférieure avec 3 icônes : Email confirmation, Calendar sync, Appointment reminder Navigation minimaliste : Services, À propos, Journal, Contact, compte, FR/EN
Architecture de l'Application 2.1 Stack technique recommandé Couche Technologie Justification Frontend React + Tailwind CSS Composants réutilisables, style utilitaire rapide Routing React Router SPA multi-pages fluide Backend Node.js + Express (ou JavaScript full-stack, Next.js API) simple Base de données SQLite ou PostgreSQL Léger pour début, scalable si besoin Auth JWT + bcrypt Authentification sécurisée admin Email Nodemailer + SMTP Confirmations et rappels automatiques Calendrier ical-generator Export .ics pour synchro calendrier client Hébergement Vercel / Railway / VPS Déploiement simple Alternative simplifiée : application full HTML/CSS/JS statique avec backend serverless (Supabase ou Firebase) si budget limité. 2.2 Pages de l'application Page Route Accès Description Landing Page / Public Page principale de réservation (hero + formulaire + calendrier) Confirmation /confirmation Public Page affichée après réservation réussie Login Admin /admin/login Public Connexion administrateur Dashboard Admin /admin/dashboard Privé (admin) Vue d'ensemble des RDV du jour/semaine Gestion RDV /admin/appointments Privé (admin) Liste complète, filtres, actions Gestion Clients /admin/clients Privé (admin) Base clients avec historique Paramètres /admin/settings Privé (admin) Horaires, services, créneaux, jours fermés Total : 3 pages publiques + 4 pages admin = 7 pages
Côté Client (Pages Publiques) 3.1 Landing Page --- Structure détaillée Header / Navigation Logo « ATELIER M. » en haut à gauche (serif, majuscules, espacé) Menu horizontal droite : Services | À propos | Journal | Contact Icône compte utilisateur + Toggle langue FR/EN Style : transparent sur le hero, fond blanc au scroll (sticky) Hero Section Image plein écran : mains manucurées sur marbre, overlay sombre léger Titre centré : « Réservez votre moment » en serif italique, grande taille Sous le titre : champ email + bouton CTA doré Widget calendrier flottant à droite du hero : Sélecteur de mois avec flèches navigation Grille jours du mois (L M M J V S D) Jours disponibles en doré, jour sélectionné avec cercle Créneaux horaires sous le calendrier (10:00, 14:30, 16:00...) Créneau sélectionné = fond doré, non disponible = grisé Section Fonctionnalités (sous le hero) 3 colonnes avec icônes fines (style line-icon, pas remplies) Email confirmation --- « Confirmation immédiate » Calendar sync --- « Synchronisation facile » Appointment reminder --- « Rappels automatiques » Footer Minimaliste : logo, liens légaux, réseaux sociaux (Instagram) Copyright + mentions légales 3.2 Parcours de réservation Le parcours client est en 4 étapes simples sur la même page : Étape Action utilisateur Système
Choisir une date Clic sur un jour Charge les créneaux disponible dans le disponibles pour ce calendrier jour
Choisir un créneau Clic sur un horaire Surligne le créneau, (10:00, 14:30...) active le formulaire
Saisir email Entre son adresse email Validation format email en temps réel
Confirmer Clic sur « Prendre Crée le RDV, envoie rendez-vous » email + fichier .ics 3.3 Page de confirmation Message de succès élégant : « Votre moment est réservé » Récapitulatif : date, heure, email Bouton « Ajouter à mon calendrier » (télécharge .ics) Bouton « Modifier / Annuler mon rendez-vous » (via lien unique par email) Lien retour à l'accueil
Côté Administrateur 4.1 Login Admin Page simple : logo + champ email + mot de passe + bouton connexion Même style visuel que le site public (cohérence de marque) Gestion erreur : message discret si identifiants incorrects Token JWT stocké en httpOnly cookie 4.2 Dashboard Vue d'ensemble rapide pour la journée : Nombre de RDV aujourd'hui / cette semaine Prochain RDV (nom/email + heure) Liste des RDV du jour avec statut (confirmé, en attente, annulé) Calendrier semaine/mois en vue compacte Notifications : nouveaux RDV, annulations 4.3 Gestion des Rendez-vous Fonctionnalité Détail Liste complète Tableau avec colonnes : Date, Heure, Client (email), Statut, Actions Filtres Par date (aujourd'hui, semaine, mois, plage custom), par statut Recherche Par email client Actions Confirmer / Annuler / Supprimer un RDV Création manuelle L'admin peut créer un RDV manuellement (téléphone, walk-in) Export Télécharger la liste en CSV 4.4 Gestion des Clients Fonctionnalité Détail Liste clients Email, nombre de visites, dernière visite, statut Fiche client Historique complet des RDV (dates, statuts) Notes L'admin peut ajouter des notes sur un client Recherche Par email ou nom (si fourni) 4.5 Paramètres Paramètre Détail Horaires d'ouverture Définir heure début/fin pour chaque jour de la semaine Durée créneau 30 min par défaut, configurable (30/45/60 min) Jours fermés Marquer des dates spécifiques comme indisponibles (vacances, fériés) Pause déjeuner Définir une plage horaire bloquée chaque jour Nombre max RDV simultanés 1 par défaut (1 seule esthéticienne) Email de notification Adresse où recevoir les alertes de nouveaux RDV Texte email confirmation Personnaliser le message envoyé au client
Fonctionnalités Techniques 5.1 Système d'emails Email Déclencheur Contenu Confirmation client Après réservation Date, heure, lien annulation, fichier .ics en pièce jointe Rappel client 24h avant le RDV Rappel avec date/heure, lien pour modifier Notification admin Nouvelle réservation Détails du RDV + lien vers le dashboard Annulation Client annule Confirmation d'annulation au client + notification admin 5.2 Synchronisation Calendrier Génération d'un fichier .ics à chaque réservation Envoyé en pièce jointe de l'email de confirmation Bouton de téléchargement sur la page de confirmation Compatible : Google Calendar, Apple Calendar, Outlook 5.3 Gestion des disponibilités Les créneaux sont calculés dynamiquement selon : horaires d'ouverture, pause déjeuner, durée du créneau, RDV déjà pris, jours fermés Un créneau réservé disparaît immédiatement du calendrier public Protection contre les doubles réservations (vérification côté serveur) 5.4 Sécurité Authentification admin par JWT (httpOnly, secure, sameSite) Rate limiting sur les endpoints publics (anti-spam) Validation des données côté client ET serveur Liens d'annulation avec token unique (UUID) --- pas d'auth requise HTTPS obligatoire en production Protection CSRF
Modèle de Données 6.1 Tables Table : appointments Champ Type Description id UUID (PK) Identifiant unique client_email VARCHAR(255) Email du client client_name VARCHAR(255) nullable Nom du client (optionnel) date DATE Date du rendez-vous time_slot TIME Heure du créneau duration INTEGER Durée en minutes (défaut 30) status ENUM confirmed / cancelled / completed / no_show cancel_token UUID Token unique pour lien d'annulation admin_notes TEXT nullable Notes internes de l'admin created_at TIMESTAMP Date de création updated_at TIMESTAMP Dernière modification Table : settings Champ Type Description id INTEGER (PK) 1 seul enregistrement opening_hours JSON { lundi: {start: '09:00', end: '18:00'}, ... } slot_duration INTEGER Durée en minutes (30/45/60) lunch_break JSON { start: '12:00', end: '13:00' } max_concurrent INTEGER Nombre max de RDV simultanés closed_dates JSON ['2026-03-15', '2026-04-01', ...] notification_email VARCHAR(255) Email admin pour les alertes confirmation_text TEXT Template email de confirmation Table : admin_users Champ Type Description id INTEGER (PK) Identifiant email VARCHAR(255) Email de connexion password_hash VARCHAR(255) Hash bcrypt du mot de passe created_at TIMESTAMP Date de création
API Endpoints 7.1 Endpoints publics Méthode Route Description GET /api/available-slots?date=YYYY-MM-DD Retourne les créneaux disponibles pour une date GET /api/available-dates?month=YYYY-MM Retourne les jours ayant des créneaux libres POST /api/book Créer un RDV { email, date, time_slot } GET /api/cancel/:token Page d'annulation (affiche les détails) POST /api/cancel/:token Confirmer l'annulation 7.2 Endpoints admin (protégés JWT) Méthode Route Description POST /api/admin/login Connexion { email, password } → JWT GET /api/admin/appointments Liste des RDV (filtres en query params) POST /api/admin/appointments Créer un RDV manuellement PATCH /api/admin/appointments/:id Modifier statut, notes DELETE /api/admin/appointments/:id Supprimer un RDV GET /api/admin/clients Liste des clients uniques GET /api/admin/clients/:email Historique d'un client GET /api/admin/settings Lire les paramètres PUT /api/admin/settings Mettre à jour les paramètres GET /api/admin/stats Stats dashboard (RDV jour/semaine/mois) GET /api/admin/export/csv Export CSV des RDV
